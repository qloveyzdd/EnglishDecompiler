<threat_model>
## Trust Boundaries
- boundary: parser output -> correction layer
  data: `ParseResult`, `ParseSpan`, `ParseSummary`
- boundary: corrected result -> IR renderer
  data: corrected spans, summary, relation kinds
- boundary: browser storage -> same-sentence replay
  data: trimmed input string, token index, manual role override

## Threat Register
- threat_id: T-04-01
  category: Integrity
  component: summary regeneration
  disposition: mitigate
  mitigation_plan: Extract `summarizeSpans()` into a shared parser helper and make both raw parse flow and corrected-result flow call the same function.
- threat_id: T-04-02
  category: Integrity
  component: corrected output surfaces
  disposition: mitigate
  mitigation_plan: Implement `applyRoleOverrides()` so corrected spans, regenerated summary, JSON, and IR all derive from one corrected `ParseResult`.
- threat_id: T-04-03
  category: Information Disclosure
  component: local persistence
  disposition: mitigate
  mitigation_plan: Scope saved overrides to the trimmed sentence key plus token index only, and never persist cross-sentence or cross-device correction rules in this phase.
</threat_model>
